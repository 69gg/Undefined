## 技术实现

### Mock 策略
- Mock Tauri API 调用 (`@tauri-apps/api/core`, `@tauri-apps/plugin-dialog`)
- 使用 `runtimeClientStub` 提供可配置的 Runtime 客户端 mock
- 支持针对每个测试定制 API 响应

### 配置分离
- 单元测试使用 `vite.config.ts`
- E2E 测试使用独立的 `vitest.e2e.config.ts`
- 允许不同的超时、环境和匹配规则

### 可扩展性
- 测试工具函数封装在 `test-fixtures.ts`
- 新增测试用例只需复制模板并修改断言
- README 提供详细的编写指南

```bash
# 运行所有测试（单元 + E2E）
npm run test:all

# 运行单个测试文件
npm run test:e2e -- connection-setup.test.tsx

# 运行匹配特定模式的测试
npm run test:e2e -- -t "发送消息"
```

## 维护建议

1. **新增功能时同步编写测试**
   - 每个新的用户流程都应该有对应的 E2E 测试
   - 参考现有测试套件的结构

2. **保持测试简洁**
   - 一个测试验证一个核心行为
   - 避免过度复杂的测试逻辑

3. **定期审查测试**
   - 删除过时的测试
   - 更新 mock 数据以反映 API 变化
   - 优化慢速测试

4. **监控测试覆盖率**
   - 使用 `vitest --coverage` 生成覆盖率报告
   - 关注关键用户路径的覆盖
